**Inputs:**  
- **State \(\Psi\):** how you encode a configuration  
- **Moves/Generators \(\mathcal{G}\):** legal transformations  
- **Invariants \(\mathcal{I}\):** quantities preserved by all moves  
- **Objective \(\Phi\):** target predicate or value (solved / key found)  
- **Lattice \(\Omega\):** constraints, topology, symmetry/quotients

**Process (12 steps):**

1) **Choose a canonical representation**  
   Pick a compact, lossless encoding of states; define hashing and equality.

2) **List generators and relations**  
   View moves as a group/semigroup action; write relations (presentation) if known.

3) **Identify invariants & monotones**  
   Things that never change (parity, cycle structure, group order) and things that can only improve (distance lower bounds).

4) **Quotient symmetries (reduce the space)**  
   Factor out equivalences (color relabels; point reflection symmetries; isomorphisms) to shrink the search.

5) **Define a potential/energy**  
   A heuristic \(E(\Psi)\) that trends down as you move toward \(\Phi\). (In RE terms: \(\Delta H<0\) locally, \(R(\Phi)\) high.)

6) **Decompose hierarchically**  
   Solve in phases on cosets or substructures (corners first → edges; high bits of \(k\) → low bits, etc.).

7) **Select a solver core** (fit to structure)
   - **Exact search:** BFS, IDA*, bidirectional, meet-in-the-middle  
   - **Algebraic:** Gröbner bases, linearization, lattice reduction  
   - **Probabilistic walks:** Pollard ρ, random self-reductions  
   - **SAT/SMT reduction:** encode constraints; let the solver rip  
   - **Homotopy/continuation:** deform a tractable instance into the real one

8) **Use admissible heuristics or provable bounds**  
   Guarantees guide pruning (God’s number bounds; baby-step/giant-step complexities).

9) **Cache & reuse (transposition tables)**  
   Memorize subproblems; exploit reusability (RE’s \(R(\Phi)\)).

10) **Exploit structure-specific tricks**  
   Normal forms, coset tables, Chinese remainder splits, endomorphisms.

11) **Detect/escape traps**  
   Cycle-finding, restart schedules, multiple heuristics in portfolio.

12) **Certify & compress the solution**  
   Minimal sequences, proof transcripts, verifiable witnesses.

## A) Apply to **Rubik’s Cube**

- **\(\Psi\)**: facelets/cubie permutation+orientation encoding.  
- **\(\mathcal{G}\)**: quarter/half turns; group \(G\) with known presentation.  
- **\(\mathcal{I}\)**: permutation parity, corner/edge twist sums, slice parity.  
- **\(\Omega\)**: cube group; quotient by color relabeling and whole-cube rotations.

**Concrete path**
1. Canonical state: cubie permutation + orientation vectors.  
2. Invariants: check you’re in the reachable subset.  
3. **Two-phase decomposition** (Kociemba):  
   - Phase 1 → subgroup \(G_1\) (orient pieces, put edges in slice).  
   - Phase 2 → solve from \(G_1\) to identity.  
4. Heuristic \(E\): pattern databases (PDBs) for corners/edges; IDA* over each phase.  
5. Bidirectional or IDA* with PDB pruning; symmetry pruning.  
6. Result: near-optimal sequences (often within a few moves of God’s number).

*(Same scheme also solves larger cubes using subgroup towers + PDBs.)*

## B) Apply to **Elliptic-Curve “Reversion”** (ECDLP)

- **Goal:** find \(k \in [0,n-1]\) such that \(Q = kP\) on an EC group of order \(n\).  
- **\(\Psi\)**: a group element (a point or a linear combination thereof).  
- **\(\mathcal{G}\)**: group operations (additions, doublings); random partitions to define a walk.  
- **\(\mathcal{I}\)**: group order \(n\); linear relation over \(\mathbb{Z}_n\) between seen points.  
- **\(\Omega\)**: cyclic (or near-cyclic) abelian group; possible endomorphisms (GLV/GLS), twists, CRT splits.

**Concrete path**
1. Representation: projective coordinates for speed; store \((a,b)\) s.t. current point \(X=aP+bQ\).  
2. Reduce symmetries: exploit cofactors; work in prime-order subgroup; use CRT if group factors are known.  
3. **Solver core options**  
   **Baby-Step/Giant-Step (BSGS)**: \(O(\sqrt{n})\) time & memory.  
   **Pollard-ρ** (with distinguished points): \(O(\sqrt{n})\) time, *low* memory; parallelizes trivially.  
   Endomorphism speedups (GLV/GLS) → reduce constant factors; for special curves use negation maps, automorphisms.  
4. Potential/energy: collision progress (birthday paradox). Use good partitioning to avoid short cycles; Brent’s or Floyd’s cycle finders.  
5. Certification: when you hit a collision \(aP+bQ = a'P+b'Q\), solve \((a-a')P = (b'-b)Q\) → recover \(k\equiv (a-a')(b'-b)^{-1}\pmod n\).

**Notes (structure-specific):**
- Index calculus breaks *finite fields* DLP but not generic ECs; hence EC security.  
- Beware weak cases: anomalous curves (Smart attack), small-subgroup cofactors, MOV/FR reductions (pairings) for special curves.

### Rubik (IDA* with PDB)
```python
def ida_star(start, goal, heuristic, neighbors):
    bound = heuristic(start)
    path = [start]
    while True:
        t = dfs(path, 0, bound, goal, heuristic, neighbors)
        if t == "FOUND": return path
        bound = t

def dfs(path, g, bound, goal, h, nbrs):
    node = path[-1]
    f = g + h(node)
    if f > bound: return f
    if node == goal: return "FOUND"
    m = +inf
    for move, child in nbrs(node):
        if len(path) > 1 and cancels(move, path[-2]): 
            continue
        path.append(child)
        t = dfs(path, g+1, bound, goal, h, nbrs)
        if t == "FOUND": return "FOUND"
        if t < m: m = t
        path.pop()
    return m
```

### ECDLP (Pollard-ρ, distinguished points)
```python
def pollard_rho(P, Q, n, is_distinguished):
    # Walk state: X = a*P + b*Q
    a, b, X = 0, 0, O
    table = {}  # map distinguished X -> (a,b)
    while True:
        a, b, X = step(a, b, X, P, Q, n)  # partitioned add/double
        if is_distinguished(X):
            if X in table:
                a2, b2 = table[X]
                num = (a - a2) % n
                den = (b2 - b) % n
                k = num * inv_mod(den, n) % n
                return k
            table[X] = (a, b)
```
